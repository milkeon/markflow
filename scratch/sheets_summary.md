## 시트: 데이터 모델

| MarkFlow 데이터 모델 |  |  |  |  |
| --- | --- | --- | --- | --- |
| 테이블명 | 타입 | 주요 컬럼 | 설명 / 제약 조건 | 비고 |
| users | 관계형 | id, email, password_hash, created_at | email UNIQUE. password_hash: bcrypt 해시 저장 | 인증 주체 |
| projects | 관계형 | id, name, owner_id (→users), deleted_at, created_at | deleted_at NULL = 활성 / 값 있음 = 소프트 삭제 | Owner 단일 모델 |
| project_members | 관계형 | project_id (→projects), user_id (→users), role | role: 'owner' | 'member'<br>PK: (project_id, user_id) | 2단계 권한 |
| canvases | JSONB | id, project_id (→projects), data (JSONB), updated_at | data = {nodes: [...], edges: [...]} 전체 직렬화<br>debounce ~2초 upsert | JSONB 단일 컬럼 |
| messages | 관계형 | id, project_id (→projects), user_id (→users), content, created_at | 채팅 이력 영속화. 재접속 시 복구 용도 | 채팅 전용 |

## 시트: 실시간 추상화

| 실시간 추상화 레이어 (CollabAPI) |  |  |  |  |
| --- | --- | --- | --- | --- |
| 인터페이스 (훅/메서드) | 1순위 구현체<br>(Socket.io) | 차선 구현체<br>(Liveblocks) | 역할 설명 | 비고 |
| useCollaboration() | useSocketCollab | useLiveblocksCollab | 컴포넌트는 이 훅만 바라봄. 내부 구현체 교체 시 UI 코드 무변경 | 핵심 추상화 포인트 |
| emitNodeChange() | socket.emit(event) | Liveblocks mutation | 노드 add / update / move / delete 이벤트 발송 |  |
| onNodeChange() | socket.on(event) | useStorage 구독 | 외부 변경 수신 → Zustand 스토어 업데이트 (last-write-wins) |  |
| presence (커서) | cursor:move broadcast | useOthers() | 멀티커서 좌표 + 이름 공유. throttle ~50ms로 트래픽 절감 | 50ms throttle |
| lock() / unlock() | socket 이벤트 | Liveblocks storage | 소프트 락 — 한 노드 한 명 편집 보장. 'OO 편집 중' UI 표시 | CRDT 대체 전략 |
| init (초기 싱크) | init 이벤트 (DB→클라) | RoomProvider 자동 | 접속 시 서버가 현재 캔버스 상태 전송. 재접속 시 동일 재수신 | 잔버그 1순위 대응 |
